- 排序字段起始值规范：所有维度表中的 `_Sort` 字段（如 `Indicator_Order`, `Metric_Sort`）起始值必须为 10，步长为 10，以便后续在现有项之间插入新项而不破坏排序。
- 度量值分层命名规范：核心计算链路严格分为 `Base Value`（纯业务逻辑，无格式/时间路由）、`Cell Value`（处理时间窗口/汇率/同比路由）、`Cell Display`（负责 FORMAT 格式化文本）和 `Cell Font Color`（负责返回 HEX 颜色代码）。
- 断开维度关联规范：矩阵的行/列维度表（如 `Dim_Media_Mix_Channel`）不与事实表建立物理关系，而是通过 `SELECTEDVALUE` 获取上下文，并在度量值中使用 `CALCULATE` + `FILTER` + `IN` 或 `CONTAINSSTRING` 进行显式筛选聚合。
- SQL 维度配置规范：在构建层级维度表（如媒体渠道）时，必须包含 `Summary_Scope`（管道符分隔的子项 ID 列表）和 `Parent_Channel_ID` 字段，以支持 DAX 端的动态汇总计算和占比分母定位。

# 任务拆分 — 变更名称

> 拆分顺序：数据源连接 → 数据清洗(PQ) → 模型设计 → 度量值开发 → 可视化构建 → 安全配置
> 每个任务 = 可独立验证的原子变更
> 每个任务必须精确到表名、度量值名称和 DAX 表达式

## 前置条件

- [ ] 数据源连接已确认可用
- [ ] 项目上下文（rules/project-context.md）已更新
- [ ] （其他依赖/配置前提）

## Task 1: 任务名

- **目标**: 一句话描述
- **涉及变更**:
  - Power Query: 查询名 — 新增/修改，做什么
  - 模型: 表名 — 新增列/关系/度量值
  - 可视化: 页面名 — 新增/修改视觉对象
- **关键 DAX**（如适用）:
  ```dax
  Measure Name = 
      CALCULATE(...)
  ```
- **关键 M 代码**（如适用）:
  ```m
  let
      Source = ...
  in
      Result
  ```
- **依赖**: 无 / Task N
- **验收标准**: 怎样算完成（具体数值或可观测行为）
- **验证方法**:
  - [ ] 度量值在空白报表中返回预期值
  - [ ] 交叉筛选行为正确
  - [ ] 性能在可接受范围内（查询 < Xs）

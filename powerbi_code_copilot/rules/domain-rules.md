# 业务领域约束

## 1. 通用业务规则

- 所有金额显示统一为人民币（CNY），精确到小数点后两位
- 百分比度量值统一显示格式：保留一位小数 + `%` 后缀
- 时间维度统一基于自然年（1月-12月），如需财年需另行定义
- 同比（YoY）/环比（MoM）计算必须处理零值和空值（避免除零错误）
- 排名度量值必须处理并列情况（RANKX 的 ties 参数）

## 2. KPI 定义标准

每个 KPI 必须明确以下属性：

| 属性 | 说明 |
|------|------|
| 名称 | 业务含义清晰的名称 |
| 公式 | 精确的计算逻辑 |
| 粒度 | 最细支持的分析粒度 |
| 基准值 | 目标值或历史基准 |
| 红绿灯规则 | 好/中/差的阈值定义 |
| 数据来源 | 源表和源列 |
| 更新频率 | 实时 / 每日 / 每周 / 每月 |

## 3. 数据质量规则

- 空值处理：明确定义 BLANK() 在每个度量值中的业务含义
- 异常值：定义各指标的合理范围，超出范围需告警
- 历史数据：定义报表展示的最大时间范围
- 数据延迟：页面上必须标注"数据更新时间"

## 4. 行业特定规则

### 零售行业
- 销售额 = 数量 x 单价 - 折扣 + 税费
- 库存周转天数 = 平均库存 / 日均销售成本
- 客单价 = 总销售额 / 订单数

### 金融行业
- 收益率计算必须考虑复利
- 风险指标遵循行业监管要求
- 金额精度要求更高（小数点后四位）

### 制造行业
- OEE = 可用率 x 表现率 x 良品率
- 生产周期需考虑工作日/非工作日

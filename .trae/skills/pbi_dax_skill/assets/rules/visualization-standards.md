# 可视化规范

## 1. 布局与设计原则

### 页面布局
- 每页视觉对象数量 ≤ 8（性能与可读性平衡）
- 采用 Z 型或 F 型阅读模式布局
- KPI 卡片放在页面顶部
- 详细图表从左到右、从上到下排列
- 预留筛选器/切片器区域（顶部或左侧）

### 色彩方案
- 使用统一的企业配色方案（定义在主题 JSON 中）
- 避免超过 5 种数据色彩
- 考虑色盲友好配色（避免纯红绿对比）
- 好的/增长用蓝绿色系，差的/下降用红橙色系
- 背景色与数据色形成足够对比度

### 字体规范
- 标题：10-14pt，加粗
- 数据标签：8-10pt
- 轴标签：8-10pt
- 全报表使用同一字体族
- 中文推荐：微软雅黑 / 思源黑体
- 英文推荐：Segoe UI / DIN

## 2. 图表选型指南

| 分析目的 | 推荐图表 | 避免使用 |
|---------|---------|---------|
| 展示趋势 | 折线图、面积图 | 饼图 |
| 比较大小 | 条形图、柱状图 | 3D 图表 |
| 占比分析 | 环形图、树状图、堆叠柱状图 | 饼图（超过 5 类） |
| 分布分析 | 直方图、箱线图、散点图 | 饼图 |
| 关联分析 | 散点图、气泡图 | 柱状图 |
| 地理分析 | 地图、填充地图 | 3D 地图（性能差） |
| KPI 展示 | KPI 卡片、仪表盘 | 复杂图表 |
| 表格明细 | 矩阵、表格 | 过多列的宽表 |

### 图表禁忌
- ❌ 禁止使用 3D 效果
- ❌ 禁止使用超过 5 个分类的饼图
- ❌ 禁止双 Y 轴（除非数据高度相关且量纲差异大）
- ❌ 禁止截断 Y 轴起点（误导读者）

## 3. 交互设计

### 切片器
- 常用筛选器（日期、部门）放置在固定位置
- 日期切片器使用日期范围类型（Between）
- 多选切片器标明"全选"/"清除"功能
- 切片器数量单页不超过 5 个

### 钻取与书签
- 钻取路径遵循业务逻辑层级（如：年 → 季 → 月 → 日）
- 书签配合按钮实现页面切换
- 工具提示页面保持简洁（1-3 个视觉对象）

### 交叉筛选
- 默认使用交叉高亮（Cross Highlight）
- 仅在明确需要时启用交叉筛选（Cross Filter）
- 大型报表考虑禁用部分交互以提升性能

## 4. 移动端适配

- 为关键报表页面创建移动布局
- 移动布局优先展示 KPI 和核心指标
- 触摸友好的按钮和切片器大小
- 避免在移动布局使用宽矩阵/表格

## 5. 可访问性

- 所有视觉对象添加 Alt Text（替代文本）
- Tab 键导航顺序合理
- 色彩对比度满足 WCAG 2.1 AA 标准
- 避免仅依赖颜色传达信息（辅以形状/图案）
